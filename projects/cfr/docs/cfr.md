# CFR アルゴリズム

CFR（Counterfactual Regret Minimization）は、**自己対戦を大量に繰り返して、後悔
（regret）を減らしていく**ことで、ナッシュ均衡（GTO）を計算する手法。
不完全情報ゲームの均衡計算の定番アルゴリズム。

CFR は**アルゴリズム（手法）**であって、ライブラリではない。ここではその手順を理解し、
[Kuhn poker](kuhn-poker.md) を対象に自分で実装する。

## 核となる4つの考え方

### ① regret（後悔）

ある情報集合で「**本当はあの行動をもっとすればよかった**」という度合い。

> 「K を持っていたとき、もっと bet していれば、これだけ得だった」= bet への regret

### ② regret matching（後悔に比例して選ぶ）

次の戦略は「**後悔が大きい行動ほど高確率で選ぶ**」。正の regret に比例して行動確率を
決める（全ての regret が 0 以下なら一様分布）。

> bet への後悔が溜まっていたら、次はもっと bet するようになる

### ③ counterfactual（反実仮想）

「**もし自分がこの情報集合に来ようとしていたら**」を仮定して価値を測る（だから
"counter-factual" ＝ 反事実）。自分がそこに来る確率は外し、相手や運でそこに来る確率で
重み付けする。

### ④ average strategy（平均戦略）← 一番の落とし穴

**毎回の戦略はナッシュ均衡に収束しない**。収束するのは「**全反復の平均戦略**」。
だから戦略を毎回積算しておき、最後に平均を出す。

## CFR のループ

```
1. 戦略を適当に初期化（例：全行動を等確率）
2. 以下を何万回も繰り返す：
   - ゲーム木を辿る
   - 各情報集合で、各行動の counterfactual regret を計算
   - regret を積算する
   - regret matching で次の戦略を更新
   - 戦略を平均に積算する
3. 平均戦略 → これがナッシュ均衡（GTO）に収束
```

「各情報集合が、それぞれ独立に後悔を最小化していく」と、ゼロサムゲームでは全体が
均衡に向かう——これが理論的な保証。

## 実装する部品

| 部品 | 役割 |
|------|------|
| **情報集合のテーブル** | 各情報集合ごとに regret と戦略の積算を保持 |
| **ゲーム木を辿る再帰関数** | check/bet/fold/call を展開し、終端で利得を計算 |
| **regret matching** | regret から次の戦略を作る |
| **平均戦略の集計** | 最後に GTO を出力 |

## 検証方法

- 平均戦略から **P1 の期待値が −1/18 に近づくか**を確認（[Kuhn poker](kuhn-poker.md) の既知解）。
- （発展）**exploitability**（最適からの搾取され度）が 0 に近づくかを測る。

## CFR の変種（将来）

最初は素の CFR を実装する。性能や収束を改善する変種があり、Stage 2 以降で検討する。

- **CFR+** … regret を 0 で下限クリップし、収束を速める。
- **MCCFR** … モンテカルロでサンプリングし、大きなゲームに対応。
- **Discounted CFR (DCFR)** … 反復ごとに過去を割り引く。TexasSolver / Desktop Postflop が採用。
